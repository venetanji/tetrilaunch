Touch layout · desktop keyboard map · drag gesture & onboarding hint · fullscreen toggle · control tokens

Retro pass: all interactive chrome (FIRE, rotate cluster, pause/fullscreen icon buttons) now uses square-ish corners, chunky 2-3px borders and hard offset shadows instead of soft glow blur — buttons visually "press into" their own shadow on :active rather than scaling. A fullscreen icon button sits next to pause in the HUD's compact top bar (see gameplay.html) and as a labeled row in the pause modal; it hides itself on platforms without a Fullscreen API (e.g. iPhone Safari in-browser), and auto-requests fullscreen from inside the Play/Start button's click gesture on coarse-pointer devices only (never outside a user gesture, never when already fullscreen or running standalone/Capacitor).

Touch layout — handedness & safe areas

Rotate cluster anchors bottom-right (right-thumb reach — an explicit design call), FIRE anchors bottom-left (left thumb). Both sit inside the safe-area inset (dashed amber ring) so notches/home-indicators never clip them, and both hug their corner so the center stays clear — the aim gesture is drag-anywhere-on-canvas. Drag-gesture onboarding hint

Touch-only. A neon finger-dot appears near the cannon (left ~25%, vertical center), presses, drags back along a curve while a ghost pull-back arc grows, then releases — looping with a pause between loops. Pure CSS transform/opacity keyframes, no JS animation loop.

Shown once on a player's very first bay (persisted settings.seenDragHint), or once per session if a bay opens and 15s pass with no shot. Dismissed for good the instant a real shot fires.
