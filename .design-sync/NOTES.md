# design-sync notes — Tetrilaunch

## This repo is OFF-SCRIPT (no converter)

- The design system is **hand-authored HTML/CSS preview cards** in `design/`
  (foundations / components / screens) — there is **no React, no component package,
  no Storybook, no dist/**. The game app (`app/`) is vanilla TypeScript + Canvas.
- **Do NOT run the design-sync converter** (`package-build.mjs` etc.) — there is nothing
  for it to bundle; it would fail on the config's `layoutNote` key by design.
- The upload layout is assembled by a small script (re-create it if lost — it just copies
  files): cards at `design/<group>/<name>.html` (their `@dsCard` first-line markers register
  them in the DS pane), `tokens/tokens.css` + `tokens/app.css` (verbatim copies of
  `app/src/styles/*.css`), a root `styles.css` (Google-Fonts `@import` + `./tokens/app.css`
  + a commented body-overflow relaxation), `README.md` (= `.design-sync/conventions.md`
  + generated card index), and the `_ds_needs_recompile` sentinel.
- **No `_ds_sync.json` is uploaded** (no converter recipe for a hand-authored layout —
  deliberate, per the skill's "omitting the sidecar is the honest choice"). Every re-sync
  therefore re-verifies all cards; at ~26 cards that's cheap.

## Remote files owned by the app — never touch

- `_ds_manifest.json`, `_adherence.oxlintrc.json`, `_ds_bundle.js` at the project root are
  (re)generated by the claude.ai/design app's self-check, not uploaded by this sync. Leave
  them out of upload plans and never delete them.

## Single source of truth + drift rule

- `app/src/styles/tokens.css` is the token source of truth; `app/src/styles/app.css` is the
  component-CSS source of truth. **Every card embeds a shared CSS template** (from `:root{`
  to `.name-input:focus{…}`, byte-identical across cards) that mirrors them — when app
  styling changes, regenerate/patch that shared block across ALL cards, not just the ones
  a commit touched. The 2026-07 sync fixed exactly this drift (pixel-retro restyle had
  refreshed only 3 of 25 cards).
- Card bodies also carry app copy (menu subtitle, howto steps with literal $ values from
  `makeBaseLevel`, pause/end/draft modal content). When `app/src/ui/screens.ts` or
  `app/src/game/level.ts` economy values change, re-mirror the affected card bodies.

## Verification

- Render check: playwright-core + the preinstalled Chromium (`/opt/pw-browsers/chromium` —
  the symlink IS the executable path). Launch with `--proxy-server=$HTTPS_PROXY` and
  `ignoreHTTPSErrors: true` or Google Fonts fail TLS through the agent proxy and everything
  falls back to system fonts (check `document.fonts.check('9px "Press Start 2P"')` per page).
- Screens are 960×540 frames; component cards are doc pages — 1280×800 viewport, fullPage
  screenshots, then visually grade every card (styled / complete / plausible).

## Known render warns

- Google Fonts glyph coverage: `▶` `⛶` `⏸` `⟲` `⟳` `💣` glyphs render from fallback fonts
  inside Press Start 2P buttons — expected, same as the shipped app.
- Card animations (loader sweep, drag-hint loop, chip--danger pulse) are static in
  screenshots — expected.

## Re-sync risks

- The remote project mirrors `design/**` paths; if cards are renamed/regrouped, the old
  remote paths must be deleted explicitly (no `_ds_sync.json` anchor → no diff will catch it;
  compare `list_files` against the bundle by hand).
- `design/screens/draft.html` (mod-draft screen) was authored 2026-07-22 from
  `screens.ts draftScreen` + real `mods.ts` data — re-mirror it if the draft flow changes.
- The conventions header (`.design-sync/conventions.md`) enumerates tokens + classes;
  re-validate them against `tokens/app.css`+`tokens.css` on every sync (grep each `--token`
  and `.class`) — a renamed app class silently invalidates the header.
- Google Fonts are loaded at runtime via `@import` in `styles.css` (`[FONT_REMOTE]`-style);
  nothing ships in `fonts/`. If claude.ai/design ever blocks remote fonts, revisit.
